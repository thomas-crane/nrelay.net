About nrelay

nrelay is a console-based client for Realm of the Mad God. By itself, it will connect a character to the Nexus and will ensure the character stays connected.

nrelay supports running plugins which can interact with the game traffic and connected clients.


nrelay was built using NodeJS as the runtime and TypeScript as the language. The gulp-cli is used to build the source code.

TypeScript is a good choice when creating larger projects because it provides type safety. NodeJS includes buffer and networking modules which are essential for the networking code used in nrelay.

Design goals

When working on the source code of nrelay, several design goals have been kept in mind to ensure the code runs in a well defined manner.

  1. Maintain backwards compatibility.

    Maintaining backwards compatibility between updates ensures that a plugin written for a certain major version of nrelay will work on any other version with the same major version.

  2. Provide as much functionality as the vanilla client.

    Providing as much functionality as possible means the bot can perform almost as many tasks as the vanilla client can. There are currently some features such as shooting and moving, but there is still a large amount of functionality which needs to be implemented.

  3. Make plugins easy to write.

    Most new features which are added are to improve the experience of developing a plugin. nrelay currently provides multiple features such as plugin interop, game resource loading and pre-built components.

What can it be used for

nrelay can be used to run one or more bots simultaneously to provide a range of functionality and services. Since nrelay is built using NodeJS, other npm modules can be included to extend functionality.

For example, Firebase by Google can be added provide a database backed. There are also modules for MySQL and MongoDB.

There are still many bots that can be built without additional modules.

  1. Trade bots.

    Plugins have access to all client inventories and can perform actions when trade packets are received.

    Stat bots.

    nrelay includes a pre-built plugin component which provides the ability to monitor players entering and leaving the map which the bot is in. This can be used to provide a similar functionality to realmeye.
    Using the information, stats like real-time player counts and most popular class can be stored or displayed.

  2. Communicating with KRelay.

    Data can be sent between nrelay and KRelay through a local web server. NodeJS provides in built modules which provide several ways to make creating a local web server very easy.
    When the web server is running, KRelay can connect to the web server to send and receive data.

With one or two additional modules from npm, the possibilities are greatly extended.

  1. Discord bots.

    A discord bot can be created inside of a plugin to provide messaging functionality to other plugins. This could be used to verify discord accounts through a PM to a bot, send messages when a guild member enters the nexus, and many other features.

    Bots with database storage.

    There are many npm modules which provide easy ways to store and retreive data. Firebase is a NoSQL database which also includes authentication and hosting which can be used in nrelay. There are npm modules for many other popular databases too.
